# Тестовое задание: раздел «Задачи» в AI-чате

## Контекст

Перед вами **работающий AI-чат**: отправка сообщений с мок-ответами
ассистента, переключение и создание диалогов, поиск, выбор модели,
копирование сообщений, «Источники» и прочее. Бэкенда нет — все данные
живут на клиенте (`lib/mock-data.ts` + React state).

**Ваша задача — добавить в приложение раздел «Задачи» (Task Manager).**
Стартовый код используйте как образец качества и стиля: структура
компонентов, типизация, дизайн-токены.

Запуск проекта описан в [README.md](./README.md).

## Обязательная часть

1. **Пункт «Задачи» в сайдбаре**, открывающий раздел задач.
   Отдельный route `/tasks` или переключение вида внутри страницы —
   на ваше усмотрение.
2. **Список задач.** У задачи есть: название, опциональное описание,
   статус («Новая» / «В работе» / «Готово») и дата создания.
3. **Действия с задачей.** Создание, редактирование, удаление
   с подтверждением, смена статуса.
4. **Фильтр и поиск.** Фильтр по статусу + поиск по названию.
5. **Сохранение в `localStorage`.** Задачи переживают перезагрузку
   страницы.
6. **UI в стиле приложения.** Используйте существующие дизайн-токены
   (`app/globals.css`, `tailwind.config.ts`) и компоненты как образец —
   раздел задач не должен выглядеть чужеродным.

## Бонусы (по желанию — только после обязательной части)

- Кнопка «В задачи» у сообщения чата — создаёт задачу из текста сообщения.
- Дедлайны у задач и сортировка по ним.
- Kanban-доска с drag-n-drop по статусам.
- Счётчик незавершённых задач у пункта «Задачи» в сайдбаре.
- Тёмная тема.

## Как мы оцениваем

В порядке убывания важности:

1. **Работающая обязательная часть.** Все шесть пунктов работают
   без явных багов и ошибок в консоли — это главный критерий.
2. **Качество кода.** На уровне стартового: маленькие компоненты,
   понятные имена, аккуратная типизация, отсутствие дублирования.
   Чистый код важнее количества фич.
3. **UX-детали.** Пустые состояния, подтверждения, поведение
   на краевых случаях, плавность взаимодействий.
4. **Процесс.** Осмысленная история коммитов и заполненный `PROCESS.md` —
   включая то, как вы работали с AI-инструментами.
5. **Бонусы.** Учитываются только при готовой обязательной части:
   две аккуратно сделанные бонус-фичи ценнее пяти сырых.

## Правила

- **Срок:** 24 часа с момента получения задания.
- **AI-инструменты:** можно и **нужно** пользоваться (Claude Code, Cursor,
  ChatGPT и т.п.). Нам интересно, как вы с ними работаете, — расскажете
  об этом в `PROCESS.md`.
- Работайте в **своей копии/форке** репозитория.
- **История коммитов обязательна.** Коммитьте по ходу работы — осмысленными
  шагами, а не одним коммитом в конце. По истории мы смотрим, как вы
  двигались к результату.
- По завершении заполните [PROCESS.md](./PROCESS.md).
